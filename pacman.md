# Pacman

# 개요

pacman은 아치 리눅스의 패키지 관리자중 하나이다.

공식 저장소에 있는 패키지들은 pacman으로 설치하나, 공식 패키지에 없을 경우 AUR을 사용해야 한다.

pacman 패키지의 확장자는 ~pkg.tar.gz이다.

## 공식 저장소
* core

 core 저장소는 아치 리눅스를 설치할때 가장 필수적인 패키지들이 있는 곳이다.
* extra

 extra 저장소는 위 core 패키지들을 제외한 대부분의 패키지가 들어있는 곳이다.
* community

 community 저장소는 AUR에서 믿을 수 있는 패키지들만 있는 곳이다.
* testing

 아직 정식적으로 출시되지 않은 불완전한 패키지들이 있는 곳이다.

그 외에도 많은 저장소가 있으나 여기까지만 서술하도록 하겠다.

# 사용법

## 패키지 설치

**기본적인 패키지를 설치하는 방법이다.** 뒤에 띄어쓰기를 하여 다른 패키지들을 추가로 적을 수 있다.
```
# pacman -S (패키지 이름)

예) sudo pacman -S discord
```
위 공식 저장소에 있는 패키지 중 다른 저장소를 사용하고 싶다면 다음과 같이 적는다.
```
# pacman -S (저장소 이름)/(패키지 이름)

예) sudo pacman -S extra/chromium
```
패키지를 다운로드만 하고 설치는 하지 않을 경우 다음과 같이 적는다.
```
# pacman -Sw (패키지 이름)
```
### 검색
pacman에서 공식 저장소에 있는 패키지를 겁색하는 방법이다.
```
$ pacman -Ss (패키지 이름)
```
다음은 pacman을 통해 이미 설치한 패키지를 검색하는 방법이다.
```
$ pacman -Qs (패키지 이름)
```

## 패키지 제거
**기본적인 패키지를 제거하는 방법이다.** 뒤에 띄어쓰기를 하여 다른 패키지들을 추가로 적을 수 있다.
```
# pacman -R (패키지 이름)

예) sudo pacman -R discord
```
패키지를 제거할때 앱의 데이터까지 모두 삭제하는 방법이다.
```
# pacman -Rn (패키지 이름)
```
더 이상 필요하지 않은 다른 패키지들을 찾아서 같이 제거하는 방법이다.
```
# pacman -Rs (패키지 이름)
```
만약 한번에 여러 패키지들을 설치한 그룹 패키지를 제거할 예정이라면 다음과 같이 적는다.
```
# pacman -Ru (패키지 이름)

예) sudo pacman -Ru gnome
```
위 2개의 명령어를 합친 명령어이다.
```
# pacman -Rsn (패키지 이름)
```
## 업데이트 및 동기화
저장소를 동기화하는 방법이다. 이 명령어를 오랫동안 사용하지 않으면 나중에 패키지를 설치할때 오류가 날 수 있다.
```
# pacman -Sy
```
설치된 패키지들의 업데이트를 확인하는 방법이다. 만약 업데이트가 있을시 업데이트를 할지 물어본다.
```
# pacman -Su
```
위 2개를 합친 명령어이다. **되도록이면 시간날때 가끔 이 명령어를 꼭 사용하도록 하자.**
```
# pacman -Syu
```
